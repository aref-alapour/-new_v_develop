Wallet idempotency / concurrency (_ez_wallet_*)
==============================================

Pipeline guards: charge_once, reserve_once, refund_insufficient_once, refund_insufficient_wallet_share, refund_conflict_once (see ez_wallet_step_is_done in theme).

Player charge/reserve keys (inc/player-wallet-settle.php):
- player_charge:{order_id}:{online_paid}
- player_reserve:{order_id}:{abs_debit}
Pre-insert: ez_wallet_charge_exists_for_order (unique_description only).
Legacy/description matching removed for charge.
Reserve guard: gateway intent + online_paid > 0 but no player_charge:* → on-hold (wallet_reserve_blocked_missing_charge).
Inserts: ez_wallet_safe_insert — never throws; logs charge_once_insert_failed / reserve_once_insert_failed on real DB errors; treats MySQL 1062/Duplicate as success.
