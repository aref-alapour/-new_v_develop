## خلاصه‌ی کلی `jdf.php`

این فایل پکیج **JDF** است (Jalali Date Functions) که مجموعه‌ی توابع استاندارد برای کار با **تاریخ و زمان شمسی (هجری شمسی / جلالی)** در PHP را فراهم می‌کند.  
این توابع در کل پروژه برای:

- نمایش تاریخ/ساعت به‌صورت شمسی و فارسی (`jdate`, `jstrftime`),
- ساخت timestamp از تاریخ شمسی (`jmktime`),
- گرفتن آرایه‌ی `getdate` برای شمسی (`jgetdate`),
- اعتبارسنجی تاریخ شمسی (`jcheckdate`),
- تبدیل بین اعداد فارسی و انگلیسی (`tr_num`),
- تبدیل تاریخ میلادی ↔ شمسی (`gregorian_to_jalali`, `jalali_to_gregorian`),
- و تبدیل اجزای تاریخ/ماه/روز/هفته به نام‌های فارسی (`jdate_words`)

استفاده می‌شود.

---

## ۱. تابع `jdate($format, $timestamp = '', $none = '', $time_zone = 'Asia/Tehran', $tr_num = 'fa')`

**نقش**: معادل تابع PHP `date()` برای **تاریخ شمسی**؛ یعنی با یک timestamp، خروجی فرمت‌شده‌ی تاریخ/زمان جلالی می‌دهد.

**پارامترها**:

- `format`: رشته‌ی فرمت، مشابه `date()` اما با حروف اضافه/سازگار با شمسی.
- `timestamp` (اختیاری): زمان به ثانیه از epoch؛ اگر خالی باشد `time()` استفاده می‌شود.
- `time_zone`: پیش‌فرض `Asia/Tehran`؛ اگر `'local'` نباشد، `date_default_timezone_set` را تنظیم می‌کند.
- `tr_num`: اگر `'fa'` باشد، اعداد خروجی را به فارسی تبدیل می‌کند؛ اگر `'en'` باشد، اعداد انگلیسی می‌مانند.

**منطق کلی**:

1. timezone را تنظیم می‌کند.
2. `ts` را از روی ورودی یا `time()` می‌سازد (با اعمال offset احتمالی `T_sec`).
3. با `date('H_i_j_n_O_P_s_w_Y', $ts)` اجزای تاریخ میلادی را می‌گیرد.
4. با `gregorian_to_jalali` آن‌ها را به سال/ماه/روز شمسی تبدیل می‌کند (`$j_y, $j_m, $j_d`).
5. روز سال (`$doy`)، کبیسه بودن (`$kab`) و ... را محاسبه می‌کند.
6. روی `format` حلقه می‌زند و طبق هر کاراکتر، خروجی مناسب را می‌سازد.

**برخی از کاراکترهای مهم فرمت** (خروجی‌شان شمسی/فارسی شده است):

- `Y` → سال شمسی چهاررقمی (مثلاً `1402`).
- `y` → دو رقم آخر سال شمسی.
- `m` → ماه دو رقمی شمسی (`01` تا `12`).
- `n` → شماره‌ ماه شمسی بدون صفر اول.
- `d` → روز دو رقمی (`01` تا `31`).
- `j` → روز بدون صفر.
- `H` / `i` / `s` → ساعت، دقیقه، ثانیه (میلادی، اما در کنار تاریخ شمسی).
- `l` → نام کامل روز هفته به فارسی (با استفاده از `jdate_words('rh')`).
- `D` → نام کوتاه روز هفته به فارسی (`jdate_words('kh')`).
- `F` → نام کامل ماه شمسی (`فروردین`, `اردیبهشت`, ...).
- `M` → نام کوتاه ماه شمسی (`فر`, `ار`, ...).
- `z` → شماره‌ی روز سال شمسی (۰ تا ۳۶۵).
- `t` → تعداد روزهای ماه جاری شمسی (۳۰/۳۱ و برای اسفند با توجه به کبیسه).
- `a` / `A` → `ق.ظ` / `ب.ظ` یا `قبل از ظهر` / `بعد از ظهر`.
- `U` → خود timestamp ثانیه‌ای.
- کاراکترهای دیگری مثل `W`, `o`, `q`, `k`, `K` هم برای هفته‌ی سال، سال “هفتگی”، شکل نوشتاری سال، درصد گذشته‌ی سال و ... پیاده‌سازی شده‌اند.

**خروجی**:

- رشته‌ی فرمت‌شده‌ی تاریخ شمسی؛ در نهایت اگر `tr_num != 'en'` باشد، با `tr_num` اعداد را فارسی می‌کند.

**مثال‌ها**:

- `jdate('Y/m/d H:i', time())` → چیزی مثل `1402/01/15 18:30`.
- `jdate('l j F Y', time())` → `سه ‌شنبه 15 فروردین 1402`.

## ۲. تابع `jstrftime($format, $timestamp = '', $none = '', $time_zone = 'Asia/Tehran', $tr_num = 'fa')`

**نقش**: معادل تابع `strftime()` برای **تاریخ شمسی**؛ فرمت‌دهی بر اساس `%`-کاراکترها.

**پارامترها** مشابه `jdate` هستند، اما:

- `format` با `%` شروع‌کننده‌ی specifierها کار می‌کند (مثل `strftime`).

**منطق کلی**:

1. timezone را ست می‌کند، timestamp را به‌دست می‌آورد.
2. تاریخ میلادی را با `date('h_H_i_j_n_s_w_Y', $ts)` می‌گیرد.
3. با `gregorian_to_jalali` به شمسی تبدیل می‌کند.
4. در حلقه، هرگاه `%` دید، کاراکتر بعدی را تحلیل می‌کند (`a`, `A`, `d`, `m`, `Y`, ...).
5. بسیاری از specifierها (`a`, `A`, `d`, `j`, `U`, `V`, `W`, `b`, `B`, `C`, `g`, `G`, `H`, `M`, `p`, `P`, `r`, `R`, `S`, `T`, `c`, `D`, `F`, `x`, ...) را معادل‌سازی کرده و خروجی‌شان بر پایه‌ی تاریخ شمسی است (با کمک `jdate_words`).

**خروجی**:

- رشته‌ی فرمت‌شده با specifierهای `strftime` اما بر بستر تقویم جلالی؛ در انتها مثل `jdate`، در صورت نیاز اعداد را فارسی می‌کند.

## ۳. تابع `jmktime($h = '', $m = '', $s = '', $jm = '', $jd = '', $jy = '', ...)`

**نقش**: معادل `mktime()` برای **دریافت تاریخ/زمان شمسی** و تبدیل آن به timestamp میلادی.

**پارامترها**:

- `$h`, `$m`, `$s`: ساعت، دقیقه، ثانیه.
- `$jm`, `$jd`, `$jy`: ماه، روز، سال شمسی.
- `$timezone`: پیش‌فرض `Asia/Tehran`.

**رفتار**:

- اگر `$h` خالی باشد → `time()` برمی‌گرداند.
- اگر فقط ساعت داده شود → مانند `mktime($h)` عمل می‌کند.
- اگر ساعت و دقیقه بدون ثانیه → `mktime($h, $m)`.
- اگر ساعت، دقیقه، ثانیه بدون تاریخ شمسی → `mktime($h, $m, $s)` (یعنی تاریخ امروز میلادی).
- اگر تاریخ شمسی هم داده شود:
  - اگر فقط ماه شمسی (`$jm`) داده شده:
    - سال و روز از `jdate('Y_j')` فعلی گرفته شده و با `jalali_to_gregorian` به میلادی تبدیل می‌شود.
  - اگر ماه و روز شمسی داده شده، ولی سال نه:
    - سال از تاریخ فعلی شمسی گرفته می‌شود.
  - اگر سال/ماه/روز شمسی کامل داده شود:
    - مستقیماً با `jalali_to_gregorian($jy, $jm, $jd)` به میلادی تبدیل می‌شود.
  - در هر حالت نهایتاً خروجی یک **UNIX timestamp میلادی** است که معادل آن تاریخ/زمان شمسی است.

این تابع برای سناریوهایی مثل ساختن timestamp از تاریخ ورودی کاربر (به شمسی) استفاده می‌شود.

## ۴. تابع `jgetdate($timestamp = '', $none = '', $timezone = 'Asia/Tehran', $tn = 'en')`

**نقش**: معادل `getdate()` برای **تاریخ شمسی**.

**رفتار**:

- اگر `timestamp` خالی باشد:
  - از `time()` استفاده می‌کند.
- سپس `jdate('F_G_i_j_l_n_s_w_Y_z', ...)` را روی آن timestamp اجرا می‌کند و نتایج را در آرایه می‌چیند:

```php
return [
  'seconds' => ثانیه,
  'minutes' => دقیقه,
  'hours'   => ساعت,
  'mday'    => روز ماه (شمسی),
  'wday'    => شماره‌ی روز هفته (۱..۷),
  'mon'     => شماره‌ی ماه شمسی,
  'year'    => سال شمسی,
  'yday'    => شماره‌ی روز سال شمسی,
  'weekday' => نام روز هفته (فارسی),
  'month'   => نام ماه شمسی (فارسی),
  0         => خود timestamp
];
```

- اگر `tn = 'fa'` باشد، بخش‌های عددی هم با `tr_num` به فارسی تبدیل می‌شوند.

## ۵. تابع `jcheckdate($jm, $jd, $jy)`

**نقش**: معادل `checkdate()` برای **اعتبارسنجی تاریخ شمسی**.

**منطق**:

- ورودی‌ها را با `tr_num` به اعداد انگلیسی تبدیل می‌کند و در `$jm, $jd, $jy` می‌ریزد.
- طول ماه (`$l_d`) را محاسبه می‌کند:
  - اگر ماه ۱۲ (اسفند) و سال کبیسه‌ی خاص نباشد → ۲۹ روز،
  - در غیر این صورت از فرمول `31 - (int)($jm / 6.5)` برای تعیین ۳۱/۳۰ استفاده می‌کند.
- اگر:
  - ماه خارج از ۱..۱۲،
  - یا روز خارج از ۱..`$l_d`,
  - یا سال < ۱ باشد → `false`،
  - در غیر این صورت → `true`.

## ۶. تابع `tr_num($str, $mod = 'en', $mf = '٫')`

**نقش**: تبدیل بین **اعداد فارسی و انگلیسی** در یک رشته.

**رفتار**:

- `mod = 'fa'`:
  - کاراکترهای `'0'..'9'` و `'.'` را به `'۰'..'۹'` و `mf` (پیش‌فرض: `٫`) تبدیل می‌کند.
- `mod != 'fa'` (مثلاً `'en'`):
  - برعکس: اعداد فارسی و `mf` را به انگلیسی و `'.'` تبدیل می‌کند.

این تابع به‌صورت گسترده در دیگر توابع استفاده می‌شود تا ورودی و خروجی‌ها با اعداد فارسی/انگلیسی سازگار شوند.

## ۷. تابع `jdate_words($array, $mod = '')`

**نقش**: تبدیل **اعداد تقویمی** به **واژه‌های فارسی** (نام ماه، نام روز هفته، سال به حروف، فصل، نماد حیوانی سال و ...).

**ورودی**:

- آرایه‌ای با کلیدهای خاص، مثلاً:
  - `['mm' => 1]` برای نام ماه (`فروردین`),
  - `['rr' => 15]` برای روز ماه به حروف (`پانزده`),
  - `['rh' => 6]` برای نام روز هفته (`جمعه`),
  - `['sh' => 1402]` برای حیوان سال (چرخه‌ی ۱۲گانه),
  - `['mb' => 1]` برای برج‌های فلکی (`حمل`),
  - `['ff' => 1..12]` برای فصل‌ها (`بهار`, `تابستان`, ...),
  - `['km' => 1..12]` برای مخفف‌های ماه (`فر`, `ار`, ...),
  - `['kh' => 0..6]` برای حروف کوتاه روز هفته (`ی`, `د`, ...),
  - `['ss'` => عدد سال/دو رقم سال] برای نوشتن سال به حروف.

**رفتار**:

- روی هر جفت `type => num` حلقه می‌زند:
  - با `tr_num`، `num` را به عدد انگلیسی تبدیل می‌کند.
  - بر اساس `type`:
    - `ss`: الگوریتمی برای نوشتن سال/عدد به حروف (ترکیب هزار، صدگان، دهگان، یکان).
    - `mm`: از آرایه‌ی نام ماه‌های شمسی (`فروردین` تا `اسفند`) استفاده می‌کند.
    - `rr`: نام روز ماه به حروف (`یک`, `دو`, ... `سی و یک`).
    - `rh`: نام کامل روز هفته (`یکشنبه` تا `شنبه`).
    - `sh`: حیوان سال (موش، گاو، پلنگ، خرگوش، ...).
    - `mb`: نام برج فلکی معادل ماه شمسی (`حمل`, `ثور`, ...).
    - `ff`: فصل (`بهار`, `تابستان`, `پاییز`, `زمستان`).
    - `km`: مخفف ماه‌ها.
    - `kh`: حروف یک‌حرفی روز هفته.
    - پیش‌فرض: خود عدد بدون تغییر.

- اگر `mod == ''`:
  - آرایه‌ی تبدیل‌شده را برمی‌گرداند.
- اگر `mod` رشته‌ای باشد (مثلاً `' '`):
  - خروجی را با `implode($mod, $array)` به یک رشته‌ی واحد با جداکننده تبدیل می‌کند.

این تابع همان ابزاری است که `jdate`/`jstrftime` برای ساخت متن‌های فارسی (نام ماه، نام روز، سال به حروف و ...) از آن استفاده می‌کنند.

## ۸. تبدیل تاریخ میلادی ↔ شمسی

### ۸.۱. تابع `gregorian_to_jalali($gy, $gm, $gd, $mod = '')`

**نقش**: تبدیل **تاریخ میلادی** (Gregorian) به **شمسی** (Jalali).

**ورودی**:

- `gy`: سال میلادی،
- `gm`: ماه میلادی (۱..۱۲)،
- `gd`: روز ماه میلادی.
- اختیاری: `mod` برای تعیین نوع خروجی.

**منطق**:

- ابتدا با `tr_num` اعداد را normalize می‌کند.
- از آرایه‌ی تجمعی روزهای ماه میلادی (`$g_d_m`) استفاده می‌کند تا تعداد کل روزها را از یک مبدأ ثابت محاسبه کند.
- سپس با استفاده از روابط ریاضی (ثابت‌ها و تقسیم‌های بخش‌پذیری بر ۳۳ و ۱۲۰۵۳ و ۱۴۶۱ و ...) سال و روزهای باقی‌مانده را به سال/ماه/روز شمسی نگاشت می‌کند.
- اگر `days < 186` باشد:
  - ماه‌های ۱ تا ۶ با ۳۱ روز؛
- در غیر این صورت:
  - ماه‌های ۷ تا ۱۲ با ۳۰ روز.

**خروجی**:

- اگر `mod == ''`:

```php
[$jy, $jm, $jd]  // سال، ماه، روز شمسی
```

- اگر `mod` رشته‌ای باشد، خروجی را به صورت رشته:
  - `"jy{mod}jm{mod}jd"`

مثلاً: `gregorian_to_jalali(2023, 4, 5, '/')` → `"1402/01/16"`.

### ۸.۲. تابع `jalali_to_gregorian($jy, $jm, $jd, $mod = '')`

**نقش**: تبدیل عکس؛ **تاریخ شمسی → میلادی**.

**ورودی**:

- `jy`: سال شمسی،
- `jm`: ماه شمسی (۱..۱۲)،
- `jd`: روز ماه شمسی.

**منطق**:

- با `tr_num` اعداد را normalize می‌کند.
- سال شمسی را با ثابت `1595` جابه‌جا کرده و تعداد کل روزها را از یک مبدأ محاسبه می‌کند (`$days`).
- سپس با استفاده از تقسیم‌ها و باقیمانده‌ها (دوره‌های ۴ ساله، ۳۳ ساله، ۴۰۰ ساله) سال میلادی (`gy`) و باقی‌مانده‌ی روزها را بدست می‌آورد.
- آرایه‌ی `sal_a` شامل تعداد روزهای ماه‌های میلادی (با درنظرگرفتن سال کبیسه) است؛ با حلقه‌ای روی آن، ماه (`gm`) و روز (`gd`) را بدست می‌آورد.

**خروجی**:

- اگر `mod == ''`:

```php
[$gy, $gm, $gd]  // سال، ماه، روز میلادی
```

- در غیر این صورت:
  - `"gy{mod}gm{mod}gd"`

مثال: `jalali_to_gregorian(1402, 1, 16, '-')` → `"2023-4-5"`.

## جمع‌بندی

- `jdf.php` یک کتابخانه‌ی مستقل و کامل برای پشتیبانی از **تقویم شمسی/جلالی در PHP** است و در پروژه‌ی شما برای نمایش تاریخ‌های شمسی (مثل `jdate('H:i', $timestamp)` در وب‌سرویس‌ها) و تبدیل بین میلادی/شمسی استفاده می‌شود.
- توابع اصلی‌ای که روزمره با آن‌ها سر و کار دارید:
  - `jdate`, `jstrftime` برای فرمت‌دهی تاریخ،
  - `jmktime`, `jgetdate`, `jcheckdate` برای ساخت/تحلیل/اعتبارسنجی تاریخ شمسی،
  - `gregorian_to_jalali` و `jalali_to_gregorian` برای تبدیل بین دو تقویم،
  - و `tr_num`/`jdate_words` برای فارسی‌سازی اعداد و نام‌ها.
- این مستند خلاصه‌ی رفتاری و استفاده‌ی هر تابع را جمع کرده تا درک و استفاده‌ی بعدی از این کتابخانه در سایر وب‌سرویس‌ها ساده‌تر شود.
