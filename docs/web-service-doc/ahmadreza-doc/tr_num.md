# تابع `tr_num`

## آدرس دقیق

- **فایل:** `wp-content/themes/escapezoom-v2/ahmadreza/jdate.php`
- **خط:** حدود ۶۹۸ تا ۷۰۵

## امضا

```php
function tr_num($str, $mod = "en", $mf = "٫")
```

- **$str:** رشته اعداد (انگلیسی یا فارسی). **$mod:** "fa" → فارسی، "en" → انگلیسی. **$mf:** جداکننده اعشار در حالت فارسی.

## کار دقیق

- اعداد ۰–۹ و نقطه را بین الفبای انگلیسی و فارسی تبدیل می‌کند.
- در jdate برای خروجی اعداد فارسی یا نرمال‌سازی ورودی استفاده می‌شود.

## محل استفاده در سایت

- **داخل jdate.php:** توسط jdate, jstrftime, jmktime, jgetdate, jcheckdate, jdate_words, gregorian_to_jalali, jalali_to_gregorian.
- در تمپلیت‌ها فراخوانی مستقیم `tr_num` یافت نشد؛ استفاده غیرمستقیم از طریق خروجی jdate با $tr_num="fa".

## برای تغییر چه کار کنیم

1. **جداکننده اعشار:** پارامتر سوم را در فراخوانی یا تعریف عوض کنید.
2. **قیمت/شماره در سایت:** برای نمایش اعداد به فارسی/انگلیسی می‌توان مستقیماً `tr_num($str, 'fa')` یا `tr_num($str, 'en')` صدا زد.

## تابع/کد مشابه در سایت

- **inc/saeed-codes.php:** تابعی مثل `persianToEnglish` برای تبدیل فارسی به انگلیسی وجود دارد؛ جهت عکس tr_num با mod فراست. برای تاریخ، jdate خودش از tr_num استفاده می‌کند.
- **ادغام:** نیازی به ادغام با persianToEnglish نیست؛ یکی برای خروجی تاریخ و یکی برای نرمال‌سازی ورودی.

## بهینه‌سازی

1. در صورت نیاز به تبدیل اعداد در بخش‌های دیگر (قیمت و ...) استفاده از همین تابع یکنواختی ایجاد می‌کند.
2. از تغییر امضا/رفتار خودداری کنید تا خروجی تاریخ در کل سایت خراب نشود.
