# شورتکد `home-discounts-event`

## آدرس دقیق

- **فایل:** `wp-content/themes/escapezoom-v2/ahmadreza/shortcodes/home-discounts-event.php`
- **خط:** ۱ تا ۱۳۹ (کل فایل)

## امضا و پارامترها

- **نام شورتکد:** `[home-discounts-event]`
- **پارامتر:** ندارد.

## کار دقیق

- امروز، فردا و پس‌فردا را با `getStartAndEndTimestamps()` برای فیلتر سانس محاسبه می‌کند (مشابه home-trend-rooms).
- با `ez_webservice(array('type' => 'sort_products_get', 'data' => ["source" => "home_discounts_event"]))` لیست محصولات «تخفیف داغ هفته» را می‌گیرد و فیلد `products` (HTML) را از JSON استخراج می‌کند.
- یک سکشن با عنوان «تخفیف داغ هفته و دارای سانس»، فیلترهای «همه / امروز / فردا / پس‌فردا»، لینک «مشاهده همه» به `/discounts/` و اسلایدر event-carousel چاپ می‌کند.
- input مخفی با `id="discount-events"` و `data-source="home_discounts_event"` برای جاوااسکریپت فیلتر استفاده می‌شود.
- خروجی با `ob_start()` و `return ob_get_clean()` برگردانده می‌شود.
- **وابستگی:** `getStartAndEndTimestamps`, `ez_webservice`.

## محل استفاده در سایت

- **ثبت شورتکد:** در `ahmadreza/shortcodes/home-discounts-event.php`.
- **استفاده:** احتمالاً در صفحه اصلی یا صفحه تخفیف‌ها در محتوا یا قالب؛ در جستجوی فایل‌های PHP تم فراخوانی مستقیم یافت نشد. برای محل دقیق در `post_content` یا قالب‌های مربوط به تخفیف جستجو کنید.

## برای تغییر چه کار کنیم

1. **عنوان و لینک:** متن «تخفیف داغ هفته» و آدرس `home_url('/discounts/')` را در همان فایل ویرایش کنید.
2. **منبع داده:** `"source" => "home_discounts_event"` را فقط در صورت تغییر سمت API عوض کنید و در هر دو جا هماهنگ کنید.
3. **کلاس/آیدی:** اگر JS فیلتر به `#discount-events` یا کلاس‌های `filter-btn`, `discount-events-btn` وابسته است، پس از تغییر قالب آن‌ها را حفظ کنید.
4. **استایل/لایه:** کلاس‌های پس‌زمینه و بخش lg (مثلاً lg:bg-slate-50) را در همان فایل عوض کنید.

## تابع/کد مشابه در سایت

- **home-trend-rooms و home-scary-cinema و ...:** الگوی یکسان: محاسبه بازه روز (در صورت نیاز)، فراخوانی `ez_webservice` با یک `source`، سکشن با عنوان + فیلتر + اسلایدر. تفاوت در منبع، عنوان و گاهی فیلتر شهر به‌جای روز.
- **ادغام:** می‌توان یک تابع مثل `ez_render_home_products_section($source, $title, $filter_type = 'schedule', $link_url = '')` نوشت و هر شورتکد فقط پارامترها را بدهد تا تکرار قالب و منطق کم شود.

## بهینه‌سازی

1. **چک پاسخ webservice:** قبل از چاپ `$discountEvent` بررسی کنید که پاسخ معتبر و غیرخالی است تا در صورت خطای API خروجی خراب یا notice نداشته باشید.
2. **کش برای لیست تخفیف:** در صورت استفاده در صفحه پرترافیک می‌توان نتیجه را برای بازه زمانی کوتاه کش کرد.
3. **یکسان‌سازی با home-trend-rooms:** استخراج بخش مشترک (سکشن + فیلتر روز + اسلایدر) در یک تمپلیت/تابع و پارامتریشدن منبع و متن‌ها.
4. **دسترسی‌پذیری:** برای دکمه‌های فیلتر و اسلایدر برچسب و نقش مناسب اضافه کنید.
