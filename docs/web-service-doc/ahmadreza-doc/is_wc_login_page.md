# تابع `is_wc_login_page`

## آدرس دقیق

- **فایل:** `wp-content/themes/escapezoom-v2/ahmadreza/init.php`
- **خط:** حدود ۵۱۰ تا ۵۱۲

## امضا

```php
function is_wc_login_page(): bool
```

## کار دقیق

- برمی‌گرداند: آیا الان در صفحه اکانت ووکامرس هستیم **و** کاربر لاگین نیست.
- معادل: `is_account_page() && !is_user_logged_in()`.
- برای مخفی کردن هدر/فوتر در صفحه لاگین/ثبت‌نام استفاده می‌شود.

## محل استفاده در سایت

| فایل | خط (تقریبی) | کاربرد |
|------|-------------|--------|
| header.php | 162 | شرط نمایش بخشی از هدر |
| footer.php | 1 | شرط نمایش فوتر |

## برای تغییر چه کار کنیم

1. در `header.php` و `footer.php` شرط `! is_wc_login_page()` یا بلوک داخل آن را ویرایش کنید.
2. اگر صفحه دیگری هم «صفحه لاگین» محسوب شود، شرط را در همین تابع گسترش دهید (مثلاً با `is_page('slug')`) یا در همان تمپلیت شرط جدا بنویسید.

## تابع/کد مشابه در سایت

- ووکامرس: `is_account_page()` فقط می‌گوید صفحه اکانت است؛ تشخیص «لاگین نیست» با `!is_user_logged_in()` روی همان صفحه است.
- **ادغام:** تابع دیگری برای همین منظور تعریف نشده؛ می‌توان به هلپر مشترک (مثلاً `app/functions/helper/woocommerce.php`) منتقل کرد.

## بهینه‌سازی

1. یک نقطه تعریف در هلپر و استفاده در header/footer.
2. نام با پیشوند تم (مثلاً `ez_is_wc_login_page`) برای وضوح وابستگی.
