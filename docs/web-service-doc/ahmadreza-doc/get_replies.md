# تابع `get_replies`

## آدرس دقیق

- **فایل:** `wp-content/themes/escapezoom-v2/ahmadreza/init.php`
- **خط:** حدود ۶۴۲ تا ۶۶۵

## امضا

```php
function get_replies($items): void
```

## کار دقیق

- آرایه‌ای از آیتم‌های کامنت (با `comment_id`, `parent`, `author_title`, `content`, `replies`) می‌گیرد.
- برای هر آیتم بلوک HTML (دایو comment، عنوان، محتوا، دکمه «پاسخ دهید») چاپ می‌کند.
- به‌صورت بازگشتی `get_replies($comment['replies'])` صدا می‌زند تا پاسخ‌های تو در تو نمایش داده شوند.
- خروجی فقط با `echo` است؛ مقداری برنمی‌گرداند.

## برای تغییر چه کار کنیم

1. **قالب HTML:** در `ahmadreza/init.php` داخل تابع بلوک‌های HTML و کلاس‌ها و دکمه «پاسخ دهید» را ویرایش کنید.
2. **ساختار داده:** اگر کلیدهای API/دیتابیس عوض شد، کلیدهای استفاده‌شده (`comment_id`, `author_title`, `content`, `replies`) را با ساختار جدید هماهنگ کنید.
3. برای اسکیپ/فیلتر می‌توان پارامتر اضافی داد یا با فیلتر وردپرس خروجی هر آیتم را تغییر داد.

## تابع/کد مشابه در سایت

- وردپرس: `wp_list_comments()` برای لیست کامنت با ساختار استاندارد؛ اینجا داده از API/سفارشی است.
- **ادغام:** منطق درخت کامنت فقط اینجا پیاده شده؛ اگر جای دیگر ساختار مشابه دارید همان تابع را استفاده کنید.

## بهینه‌سازی

1. خروجی به‌صورت رشته و `return` تا در AJAX/shortcode قابل استفاده و تست باشد؛ در محل فراخوانی `echo get_replies(...)`.
2. استفاده از `esc_html()` یا `wp_kses_post()` برای همه خروجی‌های متنی (XSS).
3. جدا کردن قالب در فایل تمپلیت (مثلاً `template-parts/comment-item.php`) و include در تابع.
