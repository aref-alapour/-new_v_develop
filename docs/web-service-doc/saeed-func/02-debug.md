## saeed_store

- **تابع در `saeed-codes.php`:** `saeed_store($val = '', $die = false)`
- **جایگزین چه تابعی شده؟**  
  جایگزین مستقیم ندارد؛ یک ابزار دیباگ اختصاصی است.

### کارایی

- با استفاده از:
  ```php
  add_option(microtime(true) * 1000, $val);
  ```
  مقدار `$val` را با یک کلید یکتا در جدول `wp_options` ذخیره می‌کند.
- اگر `$die` برابر `true` باشد، بلافاصله `die()` فراخوانی می‌شود.
- در چند جای پروژه برای dump‌کردن ساختارهای بزرگ بدون چاپ روی صفحه استفاده شده (مثلاً در تحلیل topsale، زرین‌پال، سیستم امتیازدهی و…).

### بهینه‌سازی‌های پیشنهادی

1. **جلوگیری از انفجار جدول options**  
   - می‌توان کلید را به صورت قابل‌جستجو ذخیره کرد، مثلاً با prefix ثابت و امکان پاک‌سازی دوره‌ای:
     ```php
     $key = 'saeed_debug_' . time();
     add_option($key, $val, '', 'no');
     ```
   - و یک اسکریپت Cron یا صفحه ادمین برای پاک‌کردن optionهای قدیمی `saeed_debug_*`.

2. **استفاده از `error_log` یا فایل لاگ اختصاصی**  
   به‌جای options، برای دیباگ سنگین، نوشتن در فایل لاگ (داخل پوشه‌ای خارج از وب) می‌تواند هم سریع‌تر و هم تمیزتر باشد.

3. **غیرفعال‌کردن در محیط production**  
   می‌توان بر اساس ثابت `WP_DEBUG` یا یک تنظیم محیطی، این تابع را در حالت production غیر فعال یا no-op کرد.

## saeed_print

- **تابع در `saeed-codes.php`:** `saeed_print($val = '', $die = false)`
- **جایگزین چه تابعی شده؟**  
  جایگزین یک `var_dump`/`print_r` ساده در کل پروژه است.

### کارایی

- مقدار `$val` را داخل `<pre>` با `print_r` چاپ می‌کند.
- اگر `$die === true` باشد، اسکریپت را متوقف می‌کند.
- در چند جای فرانت/ادمین خروجی ایجاد می‌کند (مثلاً `content-product.php`، `saeed-codes.php` در حین توسعه).

### بهینه‌سازی‌های پیشنهادی

1. **محافظت در برابر نمایش روی سایت زنده**  
   - شرط بگذار که فقط برای کاربر ادمین یا وقتی `WP_DEBUG` روشن است کار کند:
     ```php
     if ( ! current_user_can('manage_options') ) return;
     ```

2. **حذف فراخوانی‌های باقی‌مانده در کد نهایی**  
   قبل از انتشار، با search کل پروژه، فراخوانی‌های `saeed_print` را یا کامنت یا حذف کن تا خروجی ناخواسته در HTML نداشته باشی.

3. **افزودن label اختیاری**  
   برای خوانایی بیشتر لاگ‌ها:
   ```php
   function saeed_print($val = '', $die = false, $label = null) { ... }
   ```

## `saeed_store`

| مورد | توضیح |
|------|--------|
| **تابع در saeed-codes.php** | `saeed_store( $val = '', $die = false )` |
| **جایگزین چه تابعی** | تابع اصلی است؛ برای ذخیرهٔ موقت در options. |
| **کارایی** | با `add_option( microtime(true) * 1000, $val )` مقدار را با کلید یکتا در جدول options ذخیره می‌کند. اگر `$die` true باشد بلافاصله `die()` می‌کند. در همین فایل، thankyou.php، callbacks و actions-points برای لاگ/دیباگ استفاده می‌شود. |
| **بهینه‌سازی** | ۱) در پروداکشن این تابع را فقط وقتی `WP_DEBUG` یا یک ثابت سفارشی روشن است فعال کن تا options پر نشود. ۲) یک تابع پاک‌کننده (مثلاً cron) برای حذف optionهای قدیمی با prefix مشخص اضافه کن. ۳) اگر `$val` آرایه/شیء است، قبل از ذخیره `maybe_serialize` استفاده کن تا در همهٔ محیط‌ها درست ذخیره شود. |
