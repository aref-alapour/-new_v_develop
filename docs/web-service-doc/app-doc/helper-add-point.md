# helper/add-point.php

**مسیر:** `wp-content/themes/escapezoom-v2/app/functions/helper/add-point.php`  
**بارگذاری:** از طریق `app/functions/init.php` (include_once).

---

## خلاصه

امتیازدهی به کاربر بر اساس «نوع اکشن» (مثلاً رزرو، نظر، ثبت‌نام). لیست نوع‌ها و امتیازها از فیلتر `points_list` قابل تغییر است؛ در نهایت یک رکورد در جدول `points` درج می‌شود.

---

## توابع و خطوط

### `add_point( $type, int $user = 0, string $description = '' ): void`

| پارامتر      | نوع   | توضیح |
|-------------|--------|--------|
| `$type`     | string | کلید نوع اکشن (مثلاً `place-order-leader`, `submit-comment`, `register`) |
| `$user`     | int    | شناسه کاربر؛ اگر 0 باشد از `get_current_user_id()` استفاده می‌شود |
| `$description` | string | توضیح اختیاری برای رکورد امتیاز |

**خطوط تقریبی:** ۱۰–۵۷.

**منطق:**
- اگر `$user == 0` → کاربر جاری.
- اگر `$user == 3325` → خروج بدون درج (استثنای هاردکد برای یک کاربر خاص).
- لیست نوع‌ها از فیلتر `points_list` گرفته می‌شود (پیش‌فرض: رزرو لیدر ۵۰، هم‌گروهی ۳۰، نظر ۳۰، کالکشن ۷۰/۲۰، ثبت‌نام ۲۰، تکمیل اطلاعات ۳۰).
- یک رکورد در جدول `points` با `user_id`, `point`, `action`, `description`, `created_at` درج می‌شود.

---

## استفاده در سایت

- فراخوانی از جاهایی که بعد از یک اکشن باید امتیاز داده شود: رزرو، ثبت نظر، ایجاد کالکشن، لایک کالکشن، ثبت‌نام، تکمیل پروفایل.
- جدول `points` باید وجود داشته باشد (معمولاً توسط migration یا اسکریپت دیگری ساخته می‌شود).

---

## نحوه تغییر

- **اضافه/تغییر نوع امتیاز:** از فیلتر `points_list` در تم یا پلاگین استفاده کنید و آرایه‌ای با کلیدهای جدید و مقدار `point` و `action` برگردانید.
- **حذف استثنای کاربر ۳۳۲۵:** خط ۱۷–۱۸ را حذف یا شرط را به یک متا/آپشن قابل تنظیم تبدیل کنید.
- **نام جدول:** الان هاردکد `points` است؛ در صورت پیشوند دیتابیس، از `$wpdb->prefix` استفاده شود.

---

## توابع/کدهای مشابه

- `app/functions/helper/user_level_system/actions-points.php`: امتیاز بر اساس هوک‌های ووکامرس/کالکشن با نقشه `$points_map` و تابع `add_new_point`. تفاوت: آنجا از هوک استفاده می‌شود و کلیدها متفاوت‌اند (مثلاً `place_order_leader` به جای `place-order-leader`).
- یکسان‌سازی نام نوع‌ها (با خط تیره یا آندرلاین) و متمرکز کردن لیست امتیازها در یک جا (مثلاً فقط فیلتر `points_list` یا فقط `$points_map`) از تداخل و دوباره‌کاری جلوگیری می‌کند.

---

## بهینه‌سازی پیشنهادی

1. کاربر استثنا (۳۳۲۵) را به تنظیمات (آپشن یا متا) منتقل کنید.
2. در صورت استفاده از پیشوند جدول، نام جدول را از `$wpdb` بگیرید.
3. قبل از `$wpdb->insert` بررسی کنید `$type` در لیست وجود دارد تا از خطای undefined index جلوگیری شود.
