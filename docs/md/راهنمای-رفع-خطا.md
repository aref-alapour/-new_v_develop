# راهنمای رفع خطای WordPress

## 🔴 مشکل چیست؟

خطای شما:
```
Fatal error: Uncaught Error: Attempt to assign property "post_content" on null
WordPress database error: Duplicate entry '0' for key 'wp_posts.PRIMARY'
```

**علت:** جدول `wp_posts` در دیتابیس دارای مشکل در `AUTO_INCREMENT` است. وردپرس نمی‌تواند پست جدید بسازد.

---

## ✅ راه حل (در phpMyAdmin)

### ⚠️ اگر خطای "Invalid default value for 'post_date'" گرفتید:

این خطا به خاطر تنظیمات MySQL است. ابتدا باید ستون‌های تاریخ را درست کنید.

### روش 1: اجرای کامل (پیشنهادی)

1. وارد phpMyAdmin شوید
2. پایگاه‌داده WordPress را انتخاب کنید
3. روی تب **SQL** کلیک کنید
4. محتوای فایل `fix-posts-complete.sql` را کپی و اجرا کنید:

```sql
-- مرحله 1: حذف ردیف‌های مشکل‌دار
DELETE FROM wp_posts WHERE ID = 0;

-- مرحله 2: رفع مشکل post_date
ALTER TABLE wp_posts 
MODIFY COLUMN post_date datetime NOT NULL DEFAULT '1000-01-01 00:00:00',
MODIFY COLUMN post_date_gmt datetime NOT NULL DEFAULT '1000-01-01 00:00:00',
MODIFY COLUMN post_modified datetime NOT NULL DEFAULT '1000-01-01 00:00:00',
MODIFY COLUMN post_modified_gmt datetime NOT NULL DEFAULT '1000-01-01 00:00:00';

-- مرحله 3: پیدا کردن حداکثر ID و تنظیم AUTO_INCREMENT
SET @max_id = (SELECT COALESCE(MAX(ID), 0) FROM wp_posts);
SET @new_ai = GREATEST(@max_id + 1, 1);
ALTER TABLE wp_posts AUTO_INCREMENT = @new_ai;
```

---

### روش 2: دستی گام به گام (اگر روش 1 کار نکرد)

**گام 1:** حذف ردیف‌های مشکل‌دار
```sql
DELETE FROM wp_posts WHERE ID = 0;
```

**گام 2:** رفع مشکل post_date (اجرای جداگانه)
```sql
ALTER TABLE wp_posts MODIFY COLUMN post_date datetime NOT NULL DEFAULT '1000-01-01 00:00:00';
```

**گام 3:** رفع مشکل post_date_gmt
```sql
ALTER TABLE wp_posts MODIFY COLUMN post_date_gmt datetime NOT NULL DEFAULT '1000-01-01 00:00:00';
```

**گام 4:** رفع مشکل post_modified
```sql
ALTER TABLE wp_posts MODIFY COLUMN post_modified datetime NOT NULL DEFAULT '1000-01-01 00:00:00';
```

**گام 5:** رفع مشکل post_modified_gmt
```sql
ALTER TABLE wp_posts MODIFY COLUMN post_modified_gmt datetime NOT NULL DEFAULT '1000-01-01 00:00:00';
```

**گام 6:** پیدا کردن حداکثر ID
```sql
SELECT MAX(ID) AS max_id FROM wp_posts;
```
عدد نمایش داده شده را یادداشت کنید (مثلاً: `12345`)

**گام 7:** تنظیم AUTO_INCREMENT
عدد مرحله قبل را +1 کنید و در دستور زیر قرار دهید:
```sql
ALTER TABLE wp_posts AUTO_INCREMENT = 12346;
```
(اگر max_id = 12345 بود، از 12346 استفاده کنید)

**گام 8:** بررسی
```sql
SHOW TABLE STATUS LIKE 'wp_posts';
```
در ستون `Auto_increment` باید عدد جدید را ببینید.

3. اگر خطا برطرف شد، مشکل حل شده است ✅

## ⚠️ نکات مهم

- قبل از اجرا، از دیتابیس بکاپ بگیرید
- اگر جدول شما پیشوند دیگری دارد (مثلاً `wp2_posts`)، `wp_posts` را با نام جدول خود جایگزین کنید
- اگر هنوز خطا دارید، ممکن است مشکل از جای دیگری باشد
